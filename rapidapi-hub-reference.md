## API Integration Patterns

### Authentication Methods
| Method | Used By | Complexity |
|--------|---------|------------|
| **API Key** (header/query) | Most RapidAPI APIs | Simple |
| **OAuth 2.0** | Google, Facebook, Twitter, GitHub | Medium |
| **Bearer Token** | OpenAI, Stripe, Slack | Simple |
| **Basic Auth** | Some legacy APIs | Simple |
| **JWT** | Auth0, Firebase, custom APIs | Medium |

### Common Response Formats
| Format | Usage |
|--------|-------|
| **JSON** | 95%+ of modern APIs |
| **XML** | Legacy/enterprise (SOAP) |
| **CSV** | Data export APIs |
| **Binary** | File download, images |
| **Server-Sent Events** | Streaming (OpenAI, etc.) |

### Pricing Models
| Model | Description | Example |
|-------|-------------|---------|
| **Free** | No cost, often rate-limited | REST Countries, PokeAPI |
| **Freemium** | Free tier + paid upgrades | OpenWeatherMap, Algolia |
| **Pay-per-use** | Charged per request/unit | Stripe (2.9%), Twilio ($0.0079/SMS) |
| **Subscription** | Monthly/annual flat fee | RapidAPI Pro, News API |
| **Tiered** | Multiple plans by volume | Most SaaS APIs |
